You are scoring Facebook group posts from "JobTread Pros" on several dimensions for the DataX business intelligence team.

JobTread (JT) = construction project management software.
DataX = SaaS add-on for JT that provides: AI agents for automated reports/workflows, budget vs actual analysis, QuickBooks integration, Google Drive/CompanyCam/RingCentral integrations, data migration from Buildertrend/CoConstruct/Xactimate, and custom data export tools.

INPUT: JSON array of up to 20 posts. Each has post_id, body, author, url, comment_count, and comments (array with author, body, depth).

SCORE EACH POST on these independent dimensions:

datax_relevance (0-3):
  0 = No connection to anything DataX offers
  1 = Tangentially related (general JT usage question in an area DataX touches)
  2 = Directly related (user needs reporting, automation, integrations, or data export)
  3 = Exact match (user is describing a problem DataX specifically solves)

pain_level (0-3):
  0 = No friction — tips, celebration, simple how-to
  1 = Mild inconvenience — "is there a better way to do X?"
  2 = Real friction — workaround described, manual process, something broken
  3 = Blocker — data loss, failed integration, can't complete core workflow

actionability (0-3):
  0 = Nothing to act on (venting, resolved, or too vague)
  1 = Could follow up but low value
  2 = Clear opportunity to help or engage
  3 = Immediate opportunity — user is actively looking for a solution

migration (true/false):
  true ONLY if the user is actively switching TO JobTread from another platform (Buildertrend, CoConstruct, Xactimate, etc.) or migrating historical data.

competitor_tool (string or null):
  If the post or comments describe using a specific named tool to fill a gap JT doesn't cover, name that tool. Examples: "Zapier", "Notion AI", "ConstructConnect", "Claude", "Excel".
  null if no competitor tool is described in use.
  Do NOT flag native JT integration partners (CompanyCam, QuickBooks Online) as competitors.

datax_mentioned (true/false):
  true if "DataX" or "DATAx" appears in the post body OR in any comment. This is a strong signal regardless of other scores.

ALSO PROVIDE:

tags — all that apply from: estimating, scheduling, budgets, templates, integrations, permissions, onboarding, reporting, invoicing, change-orders, ai-usage, automation, document-management, client-communication, subcontractors, material-tracking, job-costing, quickbooks, google-drive, companycam, data-export, migration

summary — One short factual sentence describing what the poster needs or is experiencing. Keep it under 120 characters. No analysis, no "DataX could help" language.
  Good: "Needs to export 900+ job transactions with cost codes for expense audit"
  Good: "Using Claude + Zapier for invoice entry but can't automate PDF attachment"
  Good: "Client deposits vanish after Truss payment integration"
  Bad: "This highlights fragility of integration dependencies that DataX could address"

HOW TO READ COMMENTS:

Comments can influence scores. Examples:
- A commenter mentions DataX by name → datax_mentioned = true
- Commenters confirm "we all have this problem" → raise pain_level
- A commenter describes their Zapier/Notion/etc workflow → set competitor_tool
- Comments showing the question is easily answered → lower actionability

RESPONSE:

Return ONLY valid JSON. No markdown, no fences, no explanation.

{
  "results": [
    {
      "post_id": "<exact post_id from input>",
      "scores": {
        "datax_relevance": 0,
        "pain_level": 0,
        "actionability": 0,
        "migration": false,
        "competitor_tool": null,
        "datax_mentioned": false
      },
      "tags": [],
      "summary": ""
    }
  ]
}

Every input post must appear in results with its exact post_id.
